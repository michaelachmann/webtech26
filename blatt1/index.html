Grundgerüst einer HTML-Seite

Projektordner, DOCTYPE, <head> und <body>.

W3Schools HTML Basic HTML Introduction
  1. Legt einen Projektordner webtech-html/ an und darin die Datei HelloWorld.html.
  2. Baut das minimale HTML5-Grundgerüst auf: 
    <!DOCTYPE html>
<html lang="de">
  <head>
    <meta charset="utf-8">
    <title>Hello World</title>
  </head>
  <body>
    <!-- Hier entsteht unser Inhalt -->
  </body>
</html>
  3. Öffnet die Datei im Browser. Der Tab-Titel sollte „Hello World“ sein — der sichtbare Inhalt ist noch leer. Warum?
  4. Ergänzt im <head> ein <meta name="author" content="EUER NAME">. Prüft in den DevTools (Rechtsklick → Untersuchen), ob das Tag korrekt im DOM erscheint.
Mini-Reflexion Notiert in einem HTML-Kommentar (<!-- ... -->) im <body>, welche Aufgaben <head> und <body> jeweils übernehmen.

Überschriften, Absätze & Textauszeichnung

Hierarchie, Semantik und erste Formatierung.

W3Schools Headings Paragraphs Formatting
  1. Erstellt eine Überschrift erster Ordnung <h1> mit dem Text „Mein erstes Web-Dokument“.
  2. Ergänzt darunter drei Absätze (<p>) über euch: Name, Studiengang, Lieblings-Webseite.
  3. Probiert die Hierarchie aus: Fügt eine <h2> „Über mich“ und eine <h3> „Interessen“ ein. Wie verändert sich die Darstellung?
  4. Hebt im Fließtext einzelne Wörter hervor — einmal mit <strong>, einmal mit <em>. Schaut euch im Browser an, wie sich die Defaults unterscheiden.
Zur Diskussion Die Tags <b> und <strong> sehen optisch identisch aus. Warum gibt es trotzdem beide? (Stichwort: strukturelles vs. semantisches Markup, vgl. Vorlesung.)

Listen

Geordnet, ungeordnet und verschachtelt.

W3Schools HTML Lists
  1. Baut unter der Überschrift „Interessen“ eine ungeordnete Liste (<ul>) mit mindestens drei Einträgen (<li>).
  2. Ergänzt darunter eine geordnete Liste (<ol>) mit drei Punkten „Was ich diese Woche lerne“.
  3. Bonus: Schachtelt eine weitere Liste innerhalb eines <li>-Elements. Was müsst ihr beachten, damit die Verschachtelung valide ist?

Links — absolut, relativ, Anker

Der Kern des HyperText: Dokumente untereinander vernetzen.

W3Schools HTML Links Bookmarks
  1. Absoluter Link: Verlinkt zur Universität (https://www.uni-regensburg.de) mit einem <a>-Tag. Ergänzt target="_blank", damit der Link in einem neuen Tab öffnet.
  2. Relativer Link: Legt eine zweite Datei about.html im gleichen Ordner an (minimales Grundgerüst aus A genügt). Verlinkt von HelloWorld.html dorthin ohne die vollständige URL — nur mit dem Dateinamen.
  3. Anker:
    • Gebt eurer <h2> „Über mich“ ein Attribut id="about".
    • Fügt ganz oben im Dokument einen Link <a href="#about">Zu mir springen</a> ein.
    • Testet das Sprungverhalten im Browser.
  4. Kombination: Verlinkt von about.html zurück nach HelloWorld.html#about — also mit Dateiname und Anker.
Hinweis Die Tabelle relativer Pfade aus der Vorlesung (same folder, child folder, parent folder…) hilft bei Teilaufgabe b) und d). Zur Not: ../ bedeutet „eine Ebene höher“.

Bilder einbauen

Medien, Pfade und Accessibility.

W3Schools HTML Images
  1. Legt einen Unterordner img/ an und speichert dort ein beliebiges Bild (z. B. katze.jpg). Achtet auf die Dateiendung.
  2. Bindet das Bild im Abschnitt „Interessen“ ein: 
    <img src="img/katze.jpg" alt="Eine schlafende Katze" width="300">
  3. Fehlersuche: Ändert den src-Pfad absichtlich auf einen nicht existierenden Dateinamen. Was zeigt der Browser an? Warum ist das alt-Attribut jetzt sichtbar? (Stichwort: Accessibility, Screen Reader.)
  4. Bindet zusätzlich ein Bild über eine absolute URL ein (z. B. das Uni-Logo von der Uni-Website). Diskutiert kurz: Wann ist welche Variante sinnvoll?

Block vs. Inline & semantische Elemente

Struktur für CSS — und für Menschen, die Screen Reader nutzen.

W3Schools Block & Inline Semantic Elements
  1. Umschließt die Bereiche „Über mich“ und „Interessen“ jeweils mit einem <section>-Tag. Ändert sich die Darstellung? (Ohne CSS: nein — die Struktur wird aber semantisch klarer.)
  2. Ergänzt ganz oben im <body> ein <header> mit der <h1>, und ganz unten ein <footer> mit einem kurzen Copyright-Hinweis.
  3. Vergleicht <div> und <span>:
    • Umschließt einen ganzen Absatz mit <div style="background: #eee">.
    • Umschließt ein einzelnes Wort in einem Absatz mit <span style="background: yellow">.
    Erklärt in eigenen Worten den Unterschied zwischen Block- und Inline-Elementen.
Hinweis Das inline-Styling (style="...") ist hier nur ein Hilfsmittel zum Sichtbarmachen. Nächste Woche lagern wir Gestaltung systematisch in CSS aus.

DevTools & DOM-Inspektion

Die Baumstruktur aus der Vorlesung am eigenen Dokument nachvollziehen.

  1. Öffnet HelloWorld.html im Browser. Rechtsklick auf die <h1> → Untersuchen. Die Entwicklertools öffnen sich mit dem Elements-Tab.
  2. Klappt den Baum auf und navigiert von <html> über <body> bis zu einem <li>-Element. Zeichnet den Pfad auf einem Notizzettel als Baum.
  3. Klickt doppelt auf den Textinhalt einer Überschrift und verändert ihn direkt im Browser. Warum ist die Änderung nach dem Neuladen wieder verschwunden?
  4. Wechselt in den Console-Tab und gebt ein: 
    document.querySelectorAll("li").length
    Was gibt der Browser zurück? Vergleicht mit der Anzahl eurer <li>-Tags.

Output vorhersagen

Notiert eure Vermutung vor dem Ausführen — und vergleicht dann.

<body>
  <p>Erster Absatz <strong>wichtig</strong> Ende.</p>
  <div>
    <h2>Titel</h2>
    <span>Inline-Text</span>
    <p>Zweiter Absatz</p>
  </div>
  <a href="#top">Nach oben</a>
</body>
  1. Welche Elemente stehen in eigenen Zeilen (Block), welche nebeneinander (Inline)?
  2. Was passiert beim Klick auf „Nach oben“, wenn kein Element die id="top" trägt?
  3. Zeichnet den zugehörigen DOM-Baum schematisch (vgl. Folie zur DOM-Darstellung).
